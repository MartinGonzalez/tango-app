Tango Instruments

Build Instruments for Tango

Tango is evolving into a developer hub where core app features and optional capabilities are delivered as Instruments. This documentation explains the foundation implemented in the current branch and how to create a new instrument project outside this repository.

Current status (v2):

Manifest-driven runtime, declarative panel contract, typed backend actions, settings schema, and `instrument-ui` are active. Tasks runs as a decoupled pilot instrument.

Mental Model

Concept Meaning
Tango Core host app and runtime for instruments.
Stage Workspace/project folder where Claude sessions run.
Instrument Installable capability with manifest, UI entrypoint and optional backend handler.
Host Panel layout + RPC + permissioned APIs exposed by Tango.

Quickstart

  1. Create a standalone JS project (outside this repo is fine).
  2. Add package.json with tango.instrument.
  3. Add SDK dependencies: @tango/instrument-sdk and optionally @tango/instrument-ui.
  4. Create entrypoints (dist/index.js and optional dist/backend.js).
  5. Frontend must export defineInstrument(...) with kind: "tango.instrument.v2".
  6. Backend should export kind: "tango.instrument.backend.v2" and actions.
  7. Run tango-sdk sync to generate tango-env.d.ts from manifest settings.
  8. Install from Tango UI via local path.
  9. Enable instrument and activate it from the Instruments launcher (hot-mount, no restart).

Minimal Example (Hello Instrument)

There are two real instrument examples in this repo:

  • developers/examples/hello-instrument (runtime minimum)
  • developers/examples/hello-ui-instrument (@tango/instrument-ui primitives)

Use them as a base to start fast.

package.json

{
  "name": "@example/tango-instrument-hello",
  "version": "0.1.0",
  "main": "dist/index.js",
  "tango": {
    "instrument": {
      "id": "hello",
      "name": "Hello",
      "group": "Examples",
      "runtime": "vanilla",
      "entrypoint": "./dist/index.js",
      "backendEntrypoint": "./dist/backend.js",
      "hostApiVersion": "2.0.0",
      "launcher": {
        "sidebarShortcut": {
          "enabled": true,
          "label": "Hello",
          "order": 40
        }
      },
      "panels": { "sidebar": true, "first": false, "second": true, "right": false },
      "permissions": ["sessions", "stages.read", "stages.observe"]
    }
  }
}

dist/index.js (frontend)

import { defineInstrument } from "@tango/instrument-sdk";

export default defineInstrument({
  kind: "tango.instrument.v2",
  defaults: {
    visible: { sidebar: true, second: true }
  },
  panels: {
    sidebar: () => ({ node: renderSidebar() }),
    second: ({ api }) => ({
      node: renderPanel(api),
      visible: true
    })
  },
  lifecycle: {
    onStop: () => cleanup()
  }
});

dist/backend.js (backend)

export default {
  kind: "tango.instrument.backend.v2",
  actions: {
    ping: {
      handler: (ctx, input) => ({
        ok: true,
        instrumentId: ctx.instrumentId,
        timestamp: new Date().toISOString(),
        input: input ?? null
      })
    }
  }
};

Frontend Lifecycle

Frontend module contract is declarative in v2:

import { defineInstrument } from "@tango/instrument-sdk";

export default defineInstrument({
  kind: "tango.instrument.v2",
  panels: {
    sidebar: ({ api }) => ({ node: renderSidebar(api) }),
    second: ({ api }) => ({ node: renderSecond(api), visible: true })
  },
  defaults: {
    visible: { sidebar: true, second: true }
  },
  lifecycle: {
    onStart: async (api) => { /* optional boot */ },
    onStop: async () => { /* optional cleanup */ }
  }
});

Return panel nodes for sidebar, first, second, and right. The host manages mounting and unmounting.

Backend Lifecycle

Backend contract is actions-based in v2:

export default {
  kind: "tango.instrument.backend.v2",
  onStart: async (ctx) => {},
  onStop: async () => {},
  actions: {
    health: {
      handler: async (ctx) => ({ ok: true, instrumentId: ctx.instrumentId })
    },
    echo: {
      input: { type: "object" },
      handler: async (_ctx, input) => ({ input })
    }
  }
};

Frontend calls backend actions through api.actions.call(name, input).

SDK Surface (v2)

  • storage: properties, files, and SQL scoped under ~/.tango/instruments/<id>/.
  • sessions: start / spawn (alias), send/kill/list/focus Claude sessions.
  • connectors: list/connect/disconnect provider auth by stage.
  • stages: list stages and get active stage.
  • events: subscribe to host hooks (session.*, snapshot.update, stage.*, etc.).
  • actions.call: typed backend action invocation.
  • settings: schema + values + secret-aware writes.
  • emit: publish instrumentEvent to frontend host.
  • @tango/instrument-sdk/react: defineReactInstrument and hooks (useInstrumentApi, useHostEvent, useInstrumentAction).

Instrument UI (@tango/instrument-ui)

@tango/instrument-ui provides vanilla DOM helpers with scoped classes (.tui-*) and Tango token fallbacks.

React wrappers are also available via @tango/instrument-ui/react.

Dependency

{
  "dependencies": {
    "@tango/instrument-ui": "file:/path/to/tango/desktop/instruments/ui"
  }
}

Frontend usage

import {
  defineInstrument,
} from "@tango/instrument-sdk";
import {
  ensureInstrumentUI,
  createRoot,
  panelHeader,
  card,
  button,
  list,
  listItem,
} from "@tango/instrument-ui";

export default defineInstrument({
  kind: "tango.instrument.v2",
  defaults: { visible: { sidebar: true, second: true } },
  panels: {
    sidebar: () => {
  ensureInstrumentUI();

  const sidebar = createRoot();
  sidebar.append(
    panelHeader({ title: "My Instrument" }),
    list({
      items: [listItem({ title: "Action", onClick: () => {} })],
    })
  );
      return { node: sidebar };
    },
    second: () => {
      const second = createRoot();
      second.append(card({ content: button({ label: "Run", variant: "primary" }) }));
      return { node: second };
    }
  }
});

Build frontend entrypoints as a bundled single-file module for runtime loading.

Install Local Instrument

  1. Open Tango and go to Instruments.
  2. Click + and choose the instrument project root directory.
  3. Enable/disable or uninstall from the instrument row.
  4. Activate the instrument from the launcher (mount happens immediately).

Hot-mount validation checklist

  1. Install developers/examples/hello-instrument.
  2. Select Hello in the instrument list.
  3. Confirm sidebar switches to instrument sidebar and second panel renders content.
  4. Click Ping backend and verify JSON response.
  5. Install developers/examples/hello-ui-instrument and validate scoped Tango-styled widgets.

Registry is stored in ~/.tango/instruments/registry.json.

Tasks Pilot Notes

  • Tasks now runs as a runtime instrument without host hardcoded view mode.
  • Core host is instrument-agnostic (no Tasks-specific RPC surface required).
  • Tasks stores data under ~/.tango/instruments/tasks/ (clean cut from legacy DB).
  • Sidebar shortcut is declared via manifest launcher metadata.

FAQ

Can I build an instrument outside this repo?

Yes. Build JS entrypoints and install from local path.

Do I need to publish to a registry now?

No. Publishing/marketplace is intentionally out of scope in this phase.

Can I access all four panels?

Manifest supports four slots. This runtime slice/demo enables sidebar + second panel for the Hello sample.
