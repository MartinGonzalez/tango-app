Tango Instruments

Build Instruments for Tango

Tango is evolving into a developer hub where core app features and optional capabilities are delivered as Instruments. This documentation explains the foundation implemented in the current branch and how to create a new instrument project outside this repository.

Current status (v1):

Runtime, manifest, permissions, registry and storage are active. Tasks is the first migrated pilot instrument. Publishing/marketplace is intentionally out of scope for now.

Mental Model

Concept Meaning
Tango Core host app and runtime for instruments.
Stage Workspace/project folder where Claude sessions run.
Instrument Installable capability with manifest, UI entrypoint and optional backend handler.
Host Panel layout + RPC + permissioned APIs exposed by Tango.

Quickstart

  1. Create a standalone JS project (outside this repo is fine).
  2. Add package.json with tango.instrument.
  3. Create JS entrypoints (dist/index.js and optional dist/backend.js).
  4. Install from Tango UI via local path.
  5. Enable instrument and activate it from the Instruments launcher (hot-mount, no restart).

Minimal Example (Hello Instrument)

There is a real, tiny instrument in this repo at developers/examples/hello-instrument.

Use it as a base to start fast.

package.json

{
  "name": "@example/tango-instrument-hello",
  "version": "0.1.0",
  "main": "dist/index.js",
  "tango": {
    "instrument": {
      "id": "hello",
      "name": "Hello",
      "group": "Examples",
      "entrypoint": "./dist/index.js",
      "backendEntrypoint": "./dist/backend.js",
      "hostApiVersion": "1.0.0",
      "panels": { "sidebar": true, "first": false, "second": true, "right": false },
      "permissions": []
    }
  }
}

dist/index.js (frontend)

let outputEl = null;

export function activate(ctx) {
  const sidebar = document.createElement("div");
  sidebar.textContent = "Hello Instrument Sidebar";

  const panel = document.createElement("div");
  const pingBtn = document.createElement("button");
  pingBtn.textContent = "Ping backend";
  outputEl = document.createElement("pre");
  outputEl.textContent = "No backend call yet.";

  pingBtn.addEventListener("click", async () => {
    const result = await ctx.invoke("ping", { source: "hello-ui" });
    outputEl.textContent = JSON.stringify(result, null, 2);
  });

  panel.append("Hello from Tango Instrument");
  panel.append(pingBtn);
  panel.append(outputEl);

  ctx.panels.mount("sidebar", sidebar);
  ctx.panels.mount("second", panel);
  ctx.panels.setVisible("sidebar", true);
  ctx.panels.setVisible("second", true);
}

export function deactivate() {
  outputEl = null;
}

dist/backend.js (backend)

export async function invoke(ctx, method, params) {
  if (method === "ping") {
    return {
      ok: true,
      instrumentId: ctx.instrumentId,
      timestamp: new Date().toISOString(),
      params: params ?? null,
    };
  }
  throw new Error(`Unknown method: ${method}`);
}

Manifest (package.json)

{
  "name": "@acme/tango-instrument-example",
  "version": "0.1.0",
  "main": "dist/index.js",
  "tango": {
    "instrument": {
      "id": "example",
      "name": "Example",
      "group": "Community",
      "entrypoint": "./dist/index.js",
      "backendEntrypoint": "./dist/backend.js",
      "hostApiVersion": "1.0.0",
      "panels": {
        "sidebar": true,
        "first": false,
        "second": true,
        "right": false
      },
      "permissions": [
        "storage.files",
        "storage.db",
        "storage.properties",
        "sessions",
        "connectors.read",
        "connectors.connect",
        "stages.read",
        "stages.observe"
      ]
    }
  }
}
  • id must be unique in the registry.
  • panels defines intended host surface visibility.
  • permissions are enforced by runtime APIs.

Frontend Lifecycle

Instrument frontend module contract:

export async function activate(ctx) {
  const root = document.createElement("div");
  root.textContent = "Hello from instrument";
  ctx.panels.mount("second", root);
  ctx.panels.setVisible("sidebar", true);
  ctx.panels.setVisible("second", true);
}

export async function deactivate() {
  // Optional cleanup
}

Use ctx.panels to mount/unmount in sidebar, first, second, or right.

Current runtime slice for docs/demo supports sidebar and second.

Backend Lifecycle

Backend module contract:

export async function invoke(ctx, method, params) {
  switch (method) {
    case "health":
      return { ok: true, instrumentId: ctx.instrumentId };
    case "echo":
      return { params };
    default:
      throw new Error(`Unknown method: ${method}`);
  }
}

The host calls this through instrumentInvoke. Keep methods explicit and versioned.

SDK Surface (v1)

  • panels: mount/unmount and visibility per panel slot.
  • storage: properties, files, and SQL scoped under ~/.tango/instruments/<id>/.
  • sessions: spawn/send/kill/list Claude sessions.
  • connectors: list/connect/disconnect provider auth by stage.
  • stages: list stages and get active stage.
  • invoke: call backend handlers through instrumentInvoke.
  • emit: publish instrumentEvent to frontend host.

Install Local Instrument

  1. Open Tango and go to Instruments.
  2. Click + and choose the instrument project root directory.
  3. Enable/disable or uninstall from the instrument row.
  4. Activate the instrument from the launcher (mount happens immediately).

Hot-mount validation checklist

  1. Install developers/examples/hello-instrument.
  2. Select Hello in the instrument list.
  3. Confirm sidebar switches to instrument sidebar and second panel renders content.
  4. Click Ping backend and verify JSON response.

Registry is stored in ~/.tango/instruments/registry.json.

Tasks Pilot Notes

  • Tasks is migrated as a first instrument with parity goals.
  • Data migration moves ~/.tango/tasks.db to instrument namespace.
  • If migration fails, only Tasks is blocked; Tango core continues running.
  • Retry is available through instrument actions.

FAQ

Can I build an instrument outside this repo?

Yes. Build JS entrypoints and install from local path.

Do I need to publish to a registry now?

No. Publishing/marketplace is intentionally out of scope in this phase.

Can I access all four panels?

Manifest supports four slots. This runtime slice/demo enables sidebar + second panel for the Hello sample.
