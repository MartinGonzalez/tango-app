---
title: UI Components Reference
description: All UI components from @tango/api with complete prop types.
---

All components are imported from `@tango/api`. They use CSS classes prefixed with `tui-` and inherit the Tango theme automatically when wrapped in `UIRoot`.

```tsx
import { UIRoot, UIPanelHeader, UISection, UICard, UIButton } from "@tango/api";
```

---

## UIRoot

Required wrapper for every panel. Injects the Tango UI stylesheet.

```ts
props: {
  className?: string;
  children?: React.ReactNode;
}
```

---

## UIPanelHeader

Standard panel header with title, optional subtitle, back button, and right-side actions.

```ts
props: {
  title: string;
  subtitle?: string;
  onBack?: () => void;
  rightActions?: React.ReactNode;
}
```

When `onBack` is provided, a back arrow button appears to the left of the title.

## UIButton

Action trigger button.

```ts
props: {
  label: string;
  variant?: "primary" | "secondary" | "danger" | "ghost";  // default: "secondary"
  size?: "sm" | "md";                                        // default: "md"
  disabled?: boolean;
  onClick?: () => void;
}
```

| Variant | Use case |
|---------|----------|
| `primary` | Main action (submit, save, create) |
| `secondary` | Secondary actions |
| `danger` | Destructive actions (delete, remove) |
| `ghost` | Subtle actions (inline, toolbar) |

---

## UIInput

Single-line text input. The `onInput` callback receives the string value directly.

```ts
props: {
  value?: string;
  placeholder?: string;
  onInput?: (value: string) => void;
}
```

:::note
Uses `onInput` (not `onChange`). The callback receives a `string`, not a React event object.
:::

## UITabs

Tab navigation with content panels.

```ts
props: {
  tabs: Array<{
    value: string;
    label: string;
    content: React.ReactNode;
  }>;
  initialValue?: string;
  onChange?: (value: string) => void;
}
```

If no `initialValue` is provided, the first tab is selected. The `onChange` callback fires with the selected tab's `value` when the user switches tabs.

## UIListItem

Clickable list item with title, subtitle, and active state.

```ts
props: {
  title: string;
  subtitle?: string;
  active?: boolean;
  onClick?: () => void;
}
```

Renders as a `<button>` when `onClick` is provided, otherwise a `<div>`.

---

## UISelectionList

```ts
  selected: string[];
  multiple?: boolean;
  onChange?: (next: string[]) => void;
}
```

---

## UIGroup

Collapsible section with header, metadata, and actions.

```ts
props: {
  title: React.ReactNode;
  subtitle?: React.ReactNode;
  expanded?: boolean;          // default: true
  active?: boolean;
  animate?: boolean;
  meta?: React.ReactNode;
  actions?: React.ReactNode;
  onToggle?: (nextExpanded: boolean) => void;
  children?: React.ReactNode;
}
```

When `onToggle` is provided, the header becomes clickable and shows a caret icon. The group body collapses/expands based on the `expanded` prop.

---

## CSS utility classes

The UI kit provides utility classes you can use directly:

| Class | Description |
|-------|-------------|
| `tui-root` | Root container (applied by `UIRoot`) |
| `tui-row` | Horizontal flex row with gap |
| `tui-col` | Vertical flex column with gap |
| `tui-spread` | Space-between flex row |
