## Risk Areas

- **Database schema change** (`workspace_path` → `stage_path`, `workspace_connectors` → `stage_connectors`): Since this is SQLite with `CREATE TABLE IF NOT EXISTS`, existing users' DBs will have the old schema. The tables are created on first run, so a fresh DB will use new names. Existing DBs will break. Since this is a clean rebrand, I'll treat it as a clean break (new data dir `~/.tango/` means fresh DBs anyway).

- **Hook installer**: The hook script at `~/.claude-sessions/hooks/` will move to `~/.tango/hooks/`. Existing installations will need to re-register.

- **Keychain entries**: `connectors-repository.ts` uses `buildKeychainAccount(workspacePath)` — the keychain account names will change, existing OAuth tokens won't transfer.

## What I will NOT rename

- Claude CLI references (it's a separate tool, the app just wraps it)
- `~/.claude/` paths (those belong to Claude CLI, not our app)
- Git remote/repo names (that's a separate decision)
- The `server/` directory or its internal code (server is a separate component that monitors Claude processes — but its package name and description will update)
